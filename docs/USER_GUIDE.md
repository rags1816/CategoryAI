# CATEGORYAI User Guide (v2.21.1)

## Orientation
- **Left rail = two collapsible workbenches** (💼 Portfolio, 📦 Category), each standalone. Click a workbench title to open its home + its minimum-path strip; ▾/▸ collapses its menu. The active workbench is tinted.
- **Minimum-path strip (top of screen, always visible):** Portfolio: `1 Charter → 2 Composition → +Deepen (Fact base & Plans) → 3 Options (P2) → 4 Report (P3)`. Category: numbered pills `1–14`. Chips: ✓ green = complete · solid petrol = you are here · plain = pending. All clickable.
- 🚦 Fresh sessions see a start screen (Portfolio / Category / Demo tour).

## Portfolio Workbench
**P1 · Charter & Composition.** Charter in four groups (all "1 · Charter — …"), Composition ("2 · Composition"), then optional deepening: "Deepen · Fact base" (21 one-line, fact-based positions in 4 banners; ⚡ fills only empty fields and lists what to validate) and "Deepen · Plans" (10 prose sections). **Inputs**: type online; **Excel round-trip** (numbers/composition, replace-or-append); **Word template round-trip** (narratives; blank or prefilled; "Label: value" lines; · guidance ignored; upload applies only content-bearing fields) with **🗂 My template language** (rename any of the 51 labels — both names accepted on upload); **source documents** (PDF/PPTX/DOCX extracted in-browser, AI-summarised, cited). **📁 My portfolios** library saves/switches complete snapshots. Every data event lands in **📜 Declarations**.

**P2 · Four stages** (each states From/Feeds): **1 Strategic questions** — AI-tailored, assumption-challenging; answers constrain generation; unanswered → "Open questions" in the report. **2 Strategic options (2–5)** — genuinely different directions incl. capability-based; edit, add your own, **adopt one or several** (adoption is always yours). **3 Weighted assessment** — scores the Stage-2 options on 8 criteria (editable weights & cells, auto-assessed from charter+fact base, advisory leader). **4 Risk heat** — 10 dimensions per option; stress-test before adopting.

**P3 · Roll-up & report.** Per line: **⚡ AI draft / ↻ Re-draft** (labelled) or **✏️ Deep-dive (manual · 1–14)** — opens the category wizard with the charter inherited, a sticky banner + sidebar chip ("updates your portfolio"), live sync, and "Return to P3". The **Report Preview** is the single download point: 📝 executive summary → ✏️ 1 · Edit & confirm (the confirmed text becomes the document of record) → 📥 2 · HTML (quick check) → ⬇ 3 · board .docx (12 sections, charts embedded: spend, Kraljic, options, risk). Portfolio risk register with owners & mitigations lives on the roll-up tab.

## Category Workbench (Steps 1–14)
Sequence matters: **Step 3 gathers evidence** (research assistant: regulatory/PESTLE signals, disruptions, price outlook) → **Step 4 synthesises** SWOT & PESTLE grounded in it (the banner shows which mode). Steps 5–10 build the evidence (both Five Forces views, Kraljic, suppliers, 5×5 risks, chessboard funnel 4 → 64-cell board → kept plays, cost drivers & levers). Step 14: **💡 Options & opportunities** (derived from driver weights × levers × kept plays; each flagged **Portfolio** — raise via P3 — or **Category** — feeds this roadmap), **🧾 Document composer** (rich/light/empty chips; include/exclude optional sections; 7Ps out by default), **⚡ Generate** (pre-flight completeness check first), then **Outputs**: Word .docx (decision-first, 2× print-crisp charts) · **📑 Business case — VM template (recommended)** and Five Case Model (both with honest `[To complete]` markers) · PowerPoint mirroring the document · Markdown/copy.

## Connecting the workbenches (always opt-in)
- Naming a category that matches a portfolio line → **reminder** banner with 👁 view-existing-strategy; continue freely.
- End of Steps 1–14 → **update** the matching line (stamped deep-dive of record) or **➕ add as a new line** (composition row + strategy stored) — both confirmed and declared. Standalone remains a valid end state.

## Data, AI, admin
Workspaces: one browser profile = one workspace; device handoff via session .json (stamped; import replaces after confirm). AI: Claude in-app / Claude API / Gemini / Sandbox (offline, simulated, labelled). Admin: API keys, local user register (label, not a lock), custom templates/archetypes, licensed chessboard plug-in slot.
